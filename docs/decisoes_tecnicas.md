# Decisões Técnicas

Este documento registra as principais decisões tomadas durante o desenvolvimento do projeto e o motivo de cada uma delas

# Frontend

## React

### Decisão

Utilizar React para desenvolver a interface.

### Motivo

- Componentização.
- Facilidade para reutilizar código.
- Atende aos requisitos do edital.
- Grande adoção no mercado.

# Build Tool

## Vite

### Decisão

Utilizar Vite para criar o projeto React.

### Motivo

- Inicialização rápida.
- Build otimizado.
- Configuração simples

# Backend

## Node.js + Express

### Decisão

Utilizar Express para construir a API REST.

### Motivo

- Simplicidade.
- Facilidade de aprendizado.
- Grande quantidade de documentação.

# Banco de Dados

## SQLite

### Decisão

Utilizar SQLite.

### Motivo

- Não necessita instalação de servidor.
- Banco em arquivo.
- Atende aos requisitos mínimos do edital.

# Comunicação

## Axios

### Decisão

Utilizar Axios para comunicação entre frontend e backend.

### Motivo

- API simples.
- Melhor tratamento de erros.
- Organização das requisições.

# Estrutura de Pastas

### Decisão

Separar frontend, backend e documentação.

### Motivo

Facilita manutenção e organização do projeto.

# API

### Decisão

Seguir padrão REST.

### Motivo

Organização das rotas e facilidade de integração.

# Interface

### Decisão

Criar uma Landing Page simples, moderna e responsiva.

### Motivo

O edital pede tanto em desktop quanto em dispositivos móveis.

# PWA

### Decisão

Adicionar Manifest e Service Worker.

### Motivo

Atender ao requisito obrigatório do processo seletivo.

# Futuras Melhorias

Caso houvesse mais tempo, poderiam ser implementados:

- Login com JWT.
- Upload real de imagens.
- Favoritos.
- Chat entre usuários.
- Dashboard administrativo.
- PostgreSQL.
